# Step by Step: canvas生成图片的跨域问题

# Step_1

有这样一个需求，有一个很长的报告页面需要生成长图，这个页面中包含有两个image标签：

<div v-for="image in images" :key="image.id">
    <img :src="image.url" :alt="image.id">
</div>
data() {
	images: [
	    {id: 1, url: 'http://example.com/a.png'},
		{id: 1, url: 'http://example.com/b.png'},
	]
}

这个报告的内容非常多，用html2canvas来试试。根据官网的介绍写出这样的代码：

const template = document.getElementById('screenshot-template') //用于生成图片的div模版
const container = document.getElementById('screenshot-container') // 生成的图片的容器

html2canvas(template).then(function(canvas) {
      let img = document.createElement('img')
      img.src = canvas.toDataURL()
      container.appendChild(img)          
})

生成一个长图成功，但是你会发现，那两个image标签，完全消失。

# Step_2

根据网上查到的解决方案，给html2canvas加一个{ useCORS: true }属性，html2canvas截图的时候，发现有image标签，会去跨域请求图片。因为canvas无法对跨域的图片进行操作，我们需要给图片加上crossOrigin="anonymous"表示跨域获取这两张图片。图片服务区后台配置允许跨域的Origin。

然而，还是会报错，图片跨域，html2canvas的js里面会去请求这两张图片，当template加载的时候，img标签拿到了这两张图片，服务端缓存了图片，但是缓存的图片没有跨域信息。所以html2canvas再去请求图片的时候，跨域了。 解决方案是请求的时候加个时间戳，这样就不会命中缓存了。

<div v-for="image in images" :key="image.id">
      <img :src="`${image.url}?${+new Date()}`" :alt="image.id" crossOrigin="anonymous">
</div>

data() {
	images: [
		{id: 1, url: 'http://example.com/a.png'},
		{id: 1, url: 'http://example.com/b.png'},
	]
}

html2canvas(template, { useCORS: true }).then(function(canvas) {
    let img = document.createElement('img')
    img.src = canvas.toDataURL()
    container.appendChild(img)
})

PC端OK了，然而，用手机一看，还是不行，因为： ￼ 支持情况并不乐观，iPhone上的iOS Safari甚至iOS13.2才开始支持。而且我的iPhone已经是iOS13.2.3了，Safari确实可以，但是微信里面自带的浏览器打开就不行，图片还是丢失。

# Step_3

换一个方案，img的src不放正常的url，先用ajax请求这两个url，拿到返回之后，转成Base64，这样html2cnavas生成图片的时候遇到这两个img标签就不会去请求图片了，而且不需要crossorigin属性了。

<div v-for="image in baseImages" :key="image">
    <img :src="image" :alt="image.id">
</div>

data() {
	images: [
		{id: 1, url: 'http://example.com/a.png'},
		{id: 1, url: 'http://example.com/b.png'},
	],
	baseImages: []
}
mounted() {
	this.setImages()
}
methods: {
    getImage(url) {
	// axios请求图片，设置格式为blob（方便FileReader对象读取），返回一个promise
      return axios({
        method: 'get',
        url: `${url}?${+new Date()}`,
        responseType: 'blob'
      })
    },
    // 图片地址转换成Base64
    transfer(blob) {
	// 有两个图片，返回promise，两个transfer都完成了才进行下一步
      return new Promise((resolve, reject) => {
        let fileReader = new FileReader()
        fileReader.onload = function(e) {
          resolve(e.target.result)
        }
        fileReader.onerror = function(e) {
          reject(e)
        }
	  // 开始读取指定的blob中的内容。一旦完成，result属性中将包含一个Base64字符表示的blob中的内容
        fileReader.readAsDataURL(blob)
      })
    },
    async setImages() {
      try {
	    // 并行执行两个请求
          axios.all([
            this.getImage(this.images[0].url),
            this.getImage(this.images[1].url)
          ])
            .then(axios.spread(async (acct, perms) => {
		    // 两个请求现在都已完成，现在开始并行转换成Base64
                const res = await Promise.all([
                  this.transfer(acct.data),
                  this.transfer(perms.data)
                ])
                if (res[0] !== '' && res[1] !== '') {
                  this.baseImages = res
                } else {
                  // do something
                }
            }))
      } catch (error) {
        console.error(error)
        // do something
      }
    }
}
